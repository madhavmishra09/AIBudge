# Economics

## Problem Context
Startups and teams overspend on SaaS tools due to redundant subscriptions, unused seats, and misfit plans. Current solutions lack transparency and defensible reasoning.

## Value Proposition
- Direct savings: Identify cheaper alternatives, credits, and unused seats.
- Indirect savings: Reduce decision-making time and improve vendor negotiations.
- Trust: Transparent audit logic with cited pricing data.

## Cost–Benefit Analysis
- Costs: Engineering time, hosting, LLM API usage, transactional email fees.
- Benefits: 10–30% SaaS spend savings per company, scalable audits, monetizable leads.

## Unit Economics
- CAC: Low via viral shareable audit links.
- LTV: Higher with premium audits/consulting upsells.
- Gross Margin: High, marginal cost = API + hosting.

## Scalability & Defensibility
- Automated audits scale with API usage.
- Proprietary audit logic + curated pricing dataset.
- Network effects: More audits → richer dataset → stronger recommendations.

## Risks & Trade-offs
- Reliance on external APIs.
- Pricing data accuracy must be maintained.
- Free vs. monetization balance.
